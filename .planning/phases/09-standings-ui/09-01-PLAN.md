# Plan 09-01: Standings UI

**Phase:** 9 — Standings UI
**Plan:** 09-01 (single plan — small scope, all fits in one pass)
**Goal:** Render division standings table on the subscribe page; highlight the subscribed team's row

---

## Objective

Consume `data.standings` from the API response (added in Phase 8) and render a division standings
table on the subscribe page. The subscribed team's row is highlighted. If `standings` is null,
the section is silently omitted. No new routes, no new files — all changes in `src/index.js`.

---

## Execution Context

- `src/index.js` — all changes here; single-file Worker

---

## Context

### API Data Shape (from Phase 8)

```json
{
  "standings": {
    "division": "U-11 Girls Premier Subdivision 7",
    "params": { "level": "Premier", "b_g": "Girls", "age": "U-11", "subdivision": 7 },
    "teams": [
      { "teamId": "8174", "name": "KRSC Strikers", "w": 0, "l": 0, "t": 1, "gf": 1, "ga": 1, "rc": 0, "pts": 1 },
      ...
    ]
  }
}
```

`standings` is `null` when discovery failed.

### Existing UI Structure (subscribe page)

```
#record-badge-area      ← record chip (e.g. "0W – 0L – 1T")
#results-section        ← scored games table
<hr class="divider">
<h2>Team Calendar</h2>
```

Standings section goes between `#results-section` and `<hr class="divider">`.

### Existing CSS Patterns to Reuse

- `.section-heading` — uppercase dimmed label, used for "Team Calendar"
- `.results-table` / `.results-table th` / `.results-table td` — table styles
- `.results-table-wrap` — wrapper with margin-top
- `var(--bg)` — neutral grey background (used for `record-value` chip)
- `var(--border)` — lighter border/separator color
- `var(--text)` / `var(--text-dim)` — text colors

### Design Decisions

- **Columns:** Team | W | L | T | Pts (5 columns). Omit GF/GA/RC — too wide for mobile, unfamiliar to soccer parents.
- **Team cell:** Show team name only (no teamId prefix). Bold + background highlight for subscribed team.
- **Division label:** Show `standings.division` as a small subtitle below the "Division Standings" heading.
- **Highlighted row:** `background: var(--bg)` + `font-weight: 700` on all cells.
- **Null handling:** Skip rendering if `data.standings` is null or `teams` is empty.

### Task 3: Add client-side JS to render standings

**Location:** In the `fetch('/api/team/...')` `.then` handler, after the results table block (~line 1383, after `document.getElementById('results-section').innerHTML = resultsHtml;` closes).

Add standings rendering block:

```js
    // ── Standings table ──
    var standings = data.standings;
    if (standings && standings.teams && standings.teams.length > 0) {
      var sHtml = '<p class="section-heading" style="margin-top:.75rem;margin-bottom:.2rem">Division Standings</p>';
      sHtml += '<p class="standings-division">' + esc(standings.division || '') + '</p>';
      sHtml += '<div class="results-table-wrap"><table class="standings-table">';
      sHtml += '<thead><tr><th>Team</th><th>W</th><th>L</th><th>T</th><th>Pts</th></tr></thead><tbody>';
      for (var si = 0; si < standings.teams.length; si++) {
        var st = standings.teams[si];
        var isSubscribed = String(st.teamId) === String(TEAM_ID);
        var rowClass = isSubscribed ? ' class="standings-highlight"' : '';
        sHtml += '<tr' + rowClass + '><td>' + esc(st.name) + '</td><td>' + st.w + '</td><td>' + st.l + '</td><td>' + st.t + '</td><td>' + st.pts + '</td></tr>';
      }
      sHtml += '</tbody></table></div>';
      document.getElementById('standings-section').innerHTML = sHtml;
    }
```
