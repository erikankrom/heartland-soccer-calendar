## Overview

Enriched the subscribe page and iCal feed with real game results, W-L-T record, and opponent context by scraping the Heartland Soccer results pages. Past games show final scores; upcoming games show opponent W-L-T record. Subscribe page displays team record, a full results table, and opponent record chips. Opponent team numbers are clickable scouting links.

## Milestone Summary

**Key Decisions:**

- Cache results under `{origin}/api/results/{teamId}` — not a real route, just a stable cache namespace to avoid collision with real routes
- `opponentRecords` built by fetching each unique opponent's results in parallel via `Promise.all`; failures return null and are omitted gracefully
- Opponent ID collection scans both `results.games` AND iCal event SUMMARYs — results.games only covers past opponents; future opponents exist only in the calendar until they appear on the results page
- Home/away opponent extraction from SUMMARY: `beforeVs` has home team number, `afterVs` has away; pick the correct side based on `isHome = beforeVs.includes(teamId)`
- Results table always visible (no toggle) — compact enough, simpler UX
- Subscribe page layout: team identity block (name, record, source) above divider; Team Calendar block (heading, subtitle, subscribe, events) below
- Grey chip (`var(--bg)`, `border-radius: 6px`) for both record value and opponent record annotation — consistent visual language

**Issues Resolved:**

- Opponent record lookup went through three iterations before correct SUMMARY regex for home+away games (date-based lookup → home-only regex → correct home+away split)
- Server-side opponent ID collection was incomplete: added iCal event SUMMARY scan after visual verification confirmed client logic was correct but data was absent
- iCal enrichment (`generateICal` game intelligence) shipped during Phase 6 without a formal Phase 7 plan

**Issues Deferred:**

- Standings lookup — league standings page has no team-ID-based URL; deferred to a future phase
- Game video feed integration — listed as active requirement for future milestone

**Technical Debt Incurred:**

- None significant; all patterns follow existing Worker conventions
