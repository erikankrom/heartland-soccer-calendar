# Milestone v1.3: Standings

**Status:** SHIPPED 2026-03-05
**Phases:** 8–9
**Total Plans:** 2

## Overview

Surface division standings on the subscribe page so users can see where their team ranks. Auto-discovered from teamId by inferring gender/age from opponent names and scanning subdivisions in parallel.

## Phases

### Phase 8: Standings Scraping

**Goal**: Scrape division standings from Heartland Soccer, enrich `/api/team/{teamId}` with standings data
**Depends on**: Phase 7 (v1.2 complete)
**Plans**: 1 plan

Plans:
- [x] 08-01: Scraping utilities + auto-discovery + API integration

**Details:**
- Research confirmed `subdiv_standings.cgi` is the endpoint; no direct teamId → standings URL mapping exists
- Auto-discovery: infer b_g (`Girls`/`Boys`) via majority vote on G/B year-suffix patterns in opponent names; infer age group (`U-11` etc.) from minimum birth year in names; scan subdivisions 1–15 in parallel via `Promise.all()`
- Error responses (`Select Subdivision Error`) distinguishable from valid pages
- HTML parsed with regex (same pattern as `team_results.cgi`): `class=text` rows, Pts in `<strong>`, RC as `&nbsp`
- Cache key: `{origin}/api/standings/{teamId}` (1-hour TTL, same `caches.default` pattern)
- `fetchStandings` runs in parallel with opponent records fetch in `handleTeamAPI`
- `standings: null` returned gracefully when inference fails
- Regex bug found in plan: word boundaries (`\b`) don't work between digits and letters in patterns like `2015G` — fixed by removing trailing `\b` and correcting 2-digit year capture

## Milestone Summary

**Key Decisions:**

- Auto-discovery via parallel 15-subdivision scan — no direct teamId→standings mapping exists anywhere on the site
- Standings cache key `{origin}/api/standings/{teamId}` — namespace-only pattern, not a real route (consistent with results cache)
- Columns: Team, W, L, T, Pts only — omit GF/GA/RC (too wide for mobile, unfamiliar to soccer parents)
- `standings: null` returned gracefully — section silently omitted in UI; no empty container
- b_g/age inference uses majority vote + minimum birth year — robust to team names that don't encode the pattern (7/9 in tested subdivision do)

**Issues Resolved:**

- `inferAge` regex word-boundary bug: `\b(20\d{2})\b` fails on `2015G` because `G` is a word char — fixed by dropping trailing `\b` and correcting `1(\d)` → `(1\d)` capture in 2-digit range pattern

**Issues Deferred:**

- Non-Premier league teams (Classic, Recreation) — only `level=Premier` valid in current season; if standings returns null, those teams silently get no standings section

**Technical Debt Incurred:**

- None significant — straight application of existing patterns
